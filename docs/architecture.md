# アーキテクチャ

## システム構成

```mermaid
graph TB
    subgraph Spreadsheet["Google Spreadsheet"]
        PL["P/L<br/>月次収入・支出データ"]
        BS["B/S<br/>初期残高・開始月"]
    end
    
    GAS["Google Apps Script<br/>Webアプリとしてデプロイ"]
    
    subgraph Worker["Cloudflare Worker"]
        Auth["Basic認証"]
        API["/api/* 処理"]
        Proxy["GAS プロキシ"]
        Static["静的アセット配信<br/>（それ以外のパス）"]
    end
    
    subgraph Frontend["Frontend (React SPA)"]
        Form["入力フォーム"]

    Spreadsheet -->|データ読み書き| GAS
    GAS -->|fetch<br/>server-to-server| Proxy

    API --> Proxy
    Worker -->|静的アセット| Static
    Frontend -->|HTTPS<br/>/api/*| Auth
    Frontend -->|静的アセット| Static
```

## データフロー

1. **P/L シート**: 月次の収入・支出を記録（手入力またはPOST API経由、0円も可）
2. **B/S シート**: 初期残高（手入力）、開始月（関数で自動取得）、月ごとの残高（手入力またはPOST API経由）を管理
3. **Google Apps Script**: スプレッドシートデータを集計してJSON APIとして公開
   - カテゴリは出現順を保持（アルファベット順ではない）
   - DateオブジェクトはYYYY-MM形式の文字列に変換
   - 総資産はB/Sシートの残高データを優先表示、未設定の月は初期残高と累積収支から算出
4. **Worker**: GAS APIへのプロキシ（Basic認証付き）、将来的にはキャッシュ層としても機能
5. **Frontend**: データを取得してグラフ表示、モーダルフォームから新規データ追加
   - 選択可能な月は開始月から先々月まで、既存データがない月のみ
   - Date文字列を正規化して処理（GASから返されるDate文字列に対応）
   - すべての入力フィールド（カテゴリ別金額、残高）は必須で初期値は0

## データ設計

### シート1: P/L（損益計算書 - 手入力 または API経由で追加）

月次の収入・支出を記録するシートです。

| カラム | 型 | 説明 |
|--------|-----|------|
| 月 | String | 対象月（YYYY-MM形式） |
| カテゴリ | String | 収支のカテゴリ（給与、食費など） |
| 金額 | Number | 正: 収入 / 負: 支出（0円も可） |

### シート2: B/S（貸借対照表 - 手入力 または API経由で追加）

初期残高、開始月、および月ごとの残高を管理するシートです。

| カラム | 型 | 説明 | 設定方法 |
|--------|-----|------|----------|
| 初期残高 | Number | 記録開始時点の総資産 | 手入力 |
| 開始月 | String | 記録開始月（YYYY-MM形式） | 関数で自動取得 |
| 月 | String | 対象月（YYYY-MM形式） | 手入力 または API経由 |
| 残高 | Number | その月の残高（円） | 手入力 または API経由 |

#### B/S シートの設定例

| A列 | B列 |
|-----|-----|
| 初期残高 | `500000`（手入力） |
| 開始月 | `=TEXT(MIN('P/L'!A:A),"YYYY-MM")` |
| 2025-01 | `630000`（手入力 または API経由） |
| 2025-02 | `750000`（手入力 または API経由） |

**注意:**
- 開始月は `P/L` シートの最小月を自動取得します
- 月ごとの残高は、フロントエンドの入力フォームからも追加・更新できます
- 残高が設定されている月は、総資産チャートでその残高が直接表示されます
- 残高が設定されていない月は、初期残高と累積収支から自動計算されます

## 残高管理と総資産計算

- **B/Sシートの構造**: B/Sシートには初期残高、開始月、および月ごとの残高が記録されます
- **残高の優先表示**: `aggregateMonthlyData`関数で、B/Sシートに残高が設定されている月はその残高を直接使用します
- **自動計算**: 残高が設定されていない月は、初期残高と累積収支から自動計算されます
- **残高の更新**: フロントエンドの入力フォームから残高を追加・更新できます

## デザインシステム

### デザインコンセプト: 陽だまりの窓

家計簿ダッシュボードは「陽だまりの窓」をコンセプトに、グラスモーフィズムと暖かい黄色系のカラーパレットで統一されています。

**デザインテーマの要素:**
- **グラス**: 柔らかく光を通す透明なガラス窓
- **光**: 黄色を基調とした暖かい陽光のグラデーション
- **浮遊**: 窓ガラスが宙に浮かぶような軽やかさ
- **繊細さ**: 窓枠の細いラインと光の境界線

### カラーパレット

#### Golden Sunlight Palette

すべてのチャートで統一された黄色系のカラーパレットを使用:

```typescript
// 単一系列チャート用（総資産推移）
PRIMARY_CHART_COLOR = '#F5B800'

// 2色パレット（収支推移）
DUAL_PALETTE = {
  positive: '#FFB74D',  // 収入 - 暖かいゴールド
  negative: '#FF8A80',  // 支出 - ソフトコーラル
  neutral: '#F9A825',   // 収支 - 深いゴールド
}

// グラデーションパレット（カテゴリ別支出 - 10色）
GRADIENT_PALETTE = [
  '#FFF9C4', '#FFE082', '#FFD54F', '#FFCA28', '#FFB74D',
  '#FFA726', '#FF9800', '#F9A825', '#F57C00', '#E65100',
]
```

### グラスモーフィズム

**カード (.glass-card)**
- 背景: グラデーション (白 → 淡いクリーム)、透明度40-25%
- ブラー: 32px、彩度180%
- ボーダー: 繊細な光の反射（上・左が明るい）
- シャドウ: 浮遊感のある影 + inset highlight
- ホバー効果: 上に2px浮き上がり、影が強調される

**ヘッダー (.glass-header)**
- 背景: グラデーション (白 → 淡いクリーム)、透明度50-30%
- ブラー: 40px、彩度180%
- ボーダー: 下部に繊細なライン
- シャドウ: inset highlight による光の表現

**背景**
- ベース: クリーム色 (#FFFDF7)
- 効果: 2つのradial-gradientで陽だまりを表現
  - 左上 (20%, 30%): 淡いクリーム
  - 右下 (80%, 70%): 淡いゴールド

### チャート設計

**視覚的統一感:**
- strokeWidth: 2.5（繊細なライン）
- グリッド opacity: 0.4（控えめ）
- Tooltip: glass効果（blur 12px、透明な背景）
- エリアチャート: グラデーション塗りで深みを表現
